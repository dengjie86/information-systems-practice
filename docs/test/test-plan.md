# 校园宿舍报修与工单管理系统 - 测试计划

## 1. 测试目标
确保系统满足学生报修、管理员派单、维修员处理的核心业务流程，验证系统在不同角色下的权限隔离与数据准确性。

## 2. 测试环境
- **浏览器**：Chrome, Edge
- **后端环境**：Spring Boot 运行环境
- **数据库**：MySQL 8.0

## 3. 测试范围
| 模块 | 测试要点 |
| :--- | :--- |
| **登录模块** | 验证学生、管理员、维修员三类角色的账号登录及权限跳转。 |
| **报修申请** | 验证学生提交报修信息的完整性、图片上传功能。 |
| **工单流转** | 验证管理员审批、派单给特定维修人的流程。 |
| **维修处理** | 验证维修员接单、反馈结果及工单状态变更。 |
| **评价系统** | 验证学生对已完成工单的评分功能。 |

## 4. 测试方法
- **黑盒测试**：按照功能需求说明书，通过 UI 界面进行功能验证。
- **流程测试**：模拟一个完整的报修周期（学生申请 -> 管理员派单 -> 维修员处理 -> 学生评价）。

## 5. 进度安排
- **初期**：编写各模块测试用例。
- **中期**：执行集成测试并记录 Bug。
- **后期**：进行回归测试，确保 Bug 已修复。
