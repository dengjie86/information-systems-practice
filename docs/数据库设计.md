# 数据库设计

数据库用的 MySQL 8.0，库名 `dorm_repair`，utf8mb4 编码。

一共 5 张表：

- `user` - 用户表，学生、管理员、维修人员都在这一张表里，用 role 区分
- `repair_category` - 故障分类，比如水电维修、门窗维修这些
- `repair_order` - 报修工单，核心表
- `repair_record` - 维修记录，维修人员接单、完成这些操作的记录
- `evaluation` - 评价，学生对工单的满意度评分

## 表之间的关系

- 一个学生能提多个工单
- 一个工单属于一个故障分类
- 一个工单可以有多条维修记录（接单、完成等）
- 一个工单最多一条评价
- 一个维修人员能处理多个工单

## 用户角色

三种角色，都在 user 表的 role 字段里：

- `STUDENT` - 学生，提交报修、查看进度、确认完成、评价
- `ADMIN` - 管理员，审核工单、分派维修人员、管理用户和分类
- `WORKER` - 维修人员，接单、处理、填维修记录

## 工单状态流转

大概就是这几个状态：

1. `PENDING_AUDIT` 待审核 - 学生刚提交
2. `PENDING_ASSIGN` 待分派 - 管理员审核通过了
3. `REJECTED` 已驳回 - 管理员觉得不合规驳回了
4. `PENDING_ACCEPT` 待接单 - 管理员分派了维修人员
5. `PROCESSING` 处理中 - 维修人员接单了
6. `PENDING_CONFIRM` 待确认 - 维修人员修完了等学生确认
7. `COMPLETED` 已完成 - 学生确认ok
8. `CLOSED` 已关闭 - 管理员手动关的

正常流程就是 1 → 2 → 4 → 5 → 6 → 7，中间管理员可以驳回变成 3。

## 主要字段说明

这里只列一下不太好从字段名看出来的字段，其他的看 schema.sql 就行：

**user 表：**
- role - STUDENT / ADMIN / WORKER
- status - 1正常 0禁用
- dorm_building / dorm_room - 只有学生需要填

**repair_order 表：**
- order_no - 工单编号，唯一，格式类似 WO202604150001
- image_url - 故障图片，多张的话逗号分隔
- priority - LOW / NORMAL / HIGH / URGENT
- assigned_worker_id - 管理员分派的时候才填

**repair_record 表：**
- action_type - ACCEPT接单 / REJECT拒单 / START开始维修 / FINISH完成
- status_before / status_after - 记录这次操作前后工单状态的变化

**evaluation 表：**
- score - 1到5分
- 一个工单只能有一条评价（order_id 加了唯一索引）

## 怎么用

按顺序跑这三个脚本就行：

```bash
mysql -u root -p < database/schema.sql
mysql -u root -p < database/init.sql
mysql -u root -p < database/mock_data.sql   # 这个是演示数据，可选
```

schema.sql 开头会先 DROP 旧表再重建，所以重复执行也没事。mock_data.sql 里面造了各种状态的工单，方便测试和演示用。
