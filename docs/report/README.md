# 中期检查材料目录

这个目录放的是中期课程检查需要的材料，按下面的顺序看就行。

## 文件清单

| 文件 | 说明 |
| --- | --- |
| `中期报告.md` | 中期主报告，按课程模板组织（基本信息 + 七章 + 附录），先看这个 |
| `系统架构.md` | 三层架构详细说明，前后端目录结构、技术选型、为什么这么选 |
| `ER图.md` | 6 张表（5 业务表 + 1 文件存储表）的关系图（ASCII）+ 字段表 + 工单状态流转图 |
| `assets/` | 报告用图（架构图、ER 图、页面截图、Git 记录截图等），按命名规范放入 |

## 配套材料

中期检查时还可能用到这几份文档：

- `../数据库设计.md` — 库表设计的口语化说明
- `../api/接口规范.md` — 全部接口列表 + 参数 + 返回示例
- `../test/测试用例.md` — 79 条测试用例
- `../test/接口测试记录.md` — 实际联调记录
- `../test/bug-list.md` — 6 个已修 Bug
- `../test/postman_collection.json` — Postman 集合
- `../../README.md` — 项目根 README，含整体进度

## 给老师 / 助教看的顺序建议

1. 仓库根目录 `README.md` 一眼看完整体进度
2. 本目录 `中期报告.md` 通读
3. 看 `ER图.md` 了解数据
4. 看 `系统架构.md` 了解技术
5. 想看测试证据就翻 `../test/` 下的几份文档
